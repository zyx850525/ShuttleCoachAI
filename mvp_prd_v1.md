# 🏸 ShuttleCoach AI · MVP PRD（功能 + 交互 + 技术方案）

> **产品愿景**：
> 让每一位羽毛球爱好者，都能拥有一个随时可用、客观理性、提供可执行建议的 **AI 羽毛球教练**。

---

## 1. 产品概述

### 1.1 产品定位

ShuttleCoach AI 是一款面向 **羽毛球爱好者（新手 – 中级）** 的 AI 技术动作分析应用。

用户上传羽毛球训练或实战视频，系统基于人体姿态识别与动作规则分析，自动输出：
- 动作评分
- 问题定位
- 可执行的改进建议

强调：**像教练一样说人话，而不是像模型一样给结论。**

---

## 2. 目标用户与使用场景

### 2.1 目标用户
- 刚入门 / 自学型羽毛球爱好者
- 有一定基础，希望提升技术稳定性的业余玩家

### 2.2 核心使用场景
- 单次训练后上传视频，快速复盘
- 想知道“动作哪里不对，但没人告诉我”

---

## 3. MVP 功能范围

### 3.1 MVP 只做一件事

> **分析一段视频，给出一页“能指导下次训练”的分析结果**

### 3.2 页面结构（MVP）

| 页面 | 说明 | 是否 MVP |
|----|----|----|
| 首页 / 视频上传 | 上传 + 提示 | ✅ |
| 分析中页面 | 进度反馈 | ✅ |
| 分析结果页 | 结论+建议，核心价值页 | ✅ |
| 历史记录 | 可选 | ❌ |
| 动作详情独立页 | 延后 | ❌ |

---

## 4. 核心交互流程

1. 用户上传羽毛球视频
2. 系统进行视频质量检测
3. AI 进行动作分析与评分
4. 输出「分析结果页」
5. 用户获得清晰的改进方向

---

## 5. 分析结果页（核心页面）

### 5.1 页面目标

让用户在 **30 秒内** 明白三件事：
1. 我整体打得怎么样
2. 哪些动作最需要改
3. 下次训练该怎么练

---

### 5.2 页面模块结构

#### ① 综合评分模块
- 综合得分（0–100）
- 等级文案（如：基础 / 进阶 / 稳定）
- 一句话总结反馈

---

#### ② 动作评分模块

支持动作（MVP）：
| 动作 | 说明 |
|---|---|
| 高远球 | 后场基础 |
| 杀球 | 进攻核心 |
| 挑球 | 防守基础 |
| 放网 | 网前基础 |

每个动作展示：
- 单项得分
- 1 句简短评价

---

#### ③ 关键问题与建议模块（最重要）

- 列出 2–4 个**最关键问题**
- 每个问题配：
  - 问题描述（人话）
  - 改进建议（可执行）

---

#### ④ 训练建议总结

- 下一次训练重点
- 推荐练习方向（不超过 3 条）

---

## 6. 技术方案（MVP）

### 6.1 系统架构

前端 → 后端 API → AI 分析引擎 → 结果生成 → 前端展示

---

### 6.2 视频上传与质量检测

**检测内容**：
- 人体是否完整入镜
- 清晰度 / 抖动
- 姿态识别成功率

不通过直接提示用户重新拍摄。

---

### 6.3 姿态识别

- 使用 MediaPipe Pose
- 提取人体关键点时序数据
- 过滤低置信度帧

---

### 6.4 动作分割与识别

- 基于关键点变化规则进行动作分割
- 轻量模型 / 规则判断动作类型

MVP 支持：杀球 / 高远球 / 挑球 / 放网

---

### 6.5 动作评分系统

- 每个动作由多个评分维度组成
- 规则加权计算 0–100 分
- 所有规则可配置、可解释

---

### 6.6 建议生成机制

1. 规则系统输出问题标签
2. 问题标签 → 建议模板
3. 可选 LLM 进行语言润色

> LLM **不参与判断，只参与表达**

---

### 6.7 多动作综合评分逻辑

- 各动作按权重参与总分
- 示例权重：
  - 杀球 25%
  - 高远球 25%
  - 挑球 20%
  - 放网 20%
  - 其他 10%

---

### 6.8 技术栈

| 层级 | 技术 |
|----|----|
| 前端 | React / Flutter |
| 后端 | FastAPI |
| AI | Python + PyTorch + MediaPipe |
| 异步任务 | Celery / Redis |
| 存储 | 对象存储 + DB |

---

## 7. MVP 成功指标

### 定性指标
- 用户是否觉得“分析说得通”
- 用户是否知道“下一步怎么练”

### 定量指标
| 指标 | 目标 |
|----|----|
| 视频分析成功率 | ≥ 70% |
| 动作识别可用率 | ≥ 65% |
| 分析结果完整阅读率 | ≥ 50% |
| 二次上传率 | ≥ 25% |

---

## 8. MVP 边界声明（非常重要）

MVP 阶段 **不追求**：
- 专业运动员级别精度
- 全自动战术理解
- 复杂 3D 建模
- 精准球轨迹还原

MVP **只追求**：
> 像一个靠谱教练一样，指出关键问题，并告诉用户下一步怎么练。
